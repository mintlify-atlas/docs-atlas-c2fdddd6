## Prerequisites

Before you begin, ensure you have the following installed:

<CardGroup cols={2}>
  <Card title="Java 25" icon="coffee">
    Required for running the Spring Boot microservices
  </Card>
  <Card title="Maven" icon="box">
    Build tool for the Java microservices
  </Card>
  <Card title="Node.js 18+" icon="node-js">
    Required for the React frontend
  </Card>
  <Card title="PostgreSQL/MySQL" icon="database">
    Databases for the microservices (or use H2 for testing)
  </Card>
</CardGroup>

<Warning>
  This system does not implement authentication or authorization. All endpoints are open for academic purposes to focus on microservices integration and Semantic Web.
</Warning>

## Installation

<Steps>
  <Step title="Clone the Repository">
    Clone the NOVA.ing Atención Médica repository to your local machine:
    
    ```bash
    git clone https://github.com/your-org/NOVA_ing-AtencionMedica.git
    cd NOVA_ing-AtencionMedica
    ```
  </Step>

  <Step title="Build All Microservices">
    Build the parent project to compile all microservices:
    
    ```bash
    mvn clean install
    ```
    
    This compiles the 5 microservices:
    - `msvc-paciente` (Patient Management)
    - `msvc-medico` (Doctor Management)
    - `msvc-cita` (Appointment Management)
    - `msvc-diagnostico` (Diagnosis Management)
    - `msvc-web-semantica` (Semantic Web Layer)
  </Step>

  <Step title="Configure Databases">
    Each microservice requires its own database. Create the following databases:
    
    <CodeGroup>
    ```sql PostgreSQL
    CREATE DATABASE db_pacientes;
    CREATE DATABASE db_medicos;
    CREATE DATABASE db_citas;
    CREATE DATABASE db_diagnosticos;
    ```
    
    ```sql MySQL
    CREATE DATABASE db_pacientes;
    CREATE DATABASE db_medicos;
    CREATE DATABASE db_citas;
    CREATE DATABASE db_diagnosticos;
    ```
    </CodeGroup>
    
    <Note>
      For development, you can use H2 in-memory database by adding the H2 dependency and configuring `application.properties` in each microservice.
    </Note>
  </Step>

  <Step title="Start Microservices">
    Open 5 separate terminal windows and start each microservice:
    
    <Tabs>
      <Tab title="msvc-medico">
        ```bash
        cd msvc-medico
        ./mvnw spring-boot:run
        ```
        Runs on: `http://localhost:8080`
      </Tab>
      
      <Tab title="msvc-cita">
        ```bash
        cd msvc-cita
        ./mvnw spring-boot:run
        ```
        Runs on: `http://localhost:8081`
      </Tab>
      
      <Tab title="msvc-diagnostico">
        ```bash
        cd msvc-diagnostico
        ./mvnw spring-boot:run
        ```
        Runs on: `http://localhost:8082`
      </Tab>
      
      <Tab title="msvc-paciente">
        ```bash
        cd msvc-paciente
        ./mvnw spring-boot:run
        ```
        Runs on: `http://localhost:8083`
      </Tab>
      
      <Tab title="msvc-web-semantica">
        ```bash
        cd msvc-web-semantica
        ./mvnw spring-boot:run
        ```
        Runs on: `http://localhost:8084`
      </Tab>
    </Tabs>
    
    <Note>
      Wait for each service to fully start before proceeding. You should see "Started Msvc...Application" in each terminal.
    </Note>
  </Step>

  <Step title="Setup Apache Jena Fuseki (Semantic Web)">
    The Semantic Web service requires Apache Jena Fuseki for RDF/SPARQL queries:
    
    ```bash
    cd msvc-web-semantica/target/apache-jena-fuseki-6.0.0
    java -jar fuseki-server.jar
    ```
    
    Then open `http://localhost:3030/` in your browser and:
    
    1. Click **"New dataset"**
    2. Set **Dataset name**: `atencion_medica`
    3. Select **Dataset type**: `Persistent (TDB2) – dataset will persist across Fuseki restarts`
    4. Click **Create dataset**
    5. In the Actions column, select **"upload data"**
    6. Upload the file: `msvc-web-semantica/src/main/resources/ontology/atencion-medica.ttl`
    7. Click **"upload now"**
  </Step>

  <Step title="Install Frontend Dependencies">
    Navigate to the frontend directory and install dependencies:
    
    ```bash
    cd nova-frontend
    npm install
    ```
  </Step>

  <Step title="Build and Run Frontend">
    Build and start the React frontend:
    
    ```bash
    npm run build
    npm run dev
    ```
    
    <Warning>
      If you encounter errors, always run `npm run build` first, then `npm run dev`.
    </Warning>
    
    The frontend will start on `http://localhost:5173` (default Vite port).
  </Step>
</Steps>

## Verify Installation

Once all services are running, verify the installation:

<AccordionGroup>
  <Accordion title="Check Microservices Health">
    Verify each microservice is running:
    
    ```bash
    # Médicos (Doctors)
    curl http://localhost:8080/medicos
    
    # Citas (Appointments)
    curl http://localhost:8081/citas
    
    # Diagnósticos (Diagnoses)
    curl http://localhost:8082/diagnosticos
    
    # Pacientes (Patients)
    curl http://localhost:8083/pacientes
    
    # Semantic Web
    curl http://localhost:8084/api/v1/semantic/health
    ```
  </Accordion>
  
  <Accordion title="Access Frontend">
    Open your browser and navigate to:
    
    ```
    http://localhost:5173
    ```
    
    You should see the NOVA.ing Atención Médica dashboard with:
    - Patient management
    - Doctor management
    - Appointment scheduling
    - Diagnosis records
    - Semantic chat interface
  </Accordion>
  
  <Accordion title="Test Semantic Queries">
    Access the Fuseki SPARQL endpoint:
    
    ```
    http://localhost:3030/atencion_medica/sparql
    ```
    
    Try a sample SPARQL query:
    
    ```sparql
    PREFIX am: <http://www.atencionmedica.org/ontology#>
    
    SELECT ?paciente ?nombre WHERE {
      ?paciente a am:Paciente .
      ?paciente am:nombre ?nombre .
    }
    LIMIT 10
    ```
  </Accordion>
</AccordionGroup>

## Troubleshooting

<AccordionGroup>
  <Accordion title="Port Already in Use">
    If a port is already in use, you can change it in each microservice's `application.properties`:
    
    ```properties
    server.port=8085
    ```
  </Accordion>
  
  <Accordion title="Database Connection Errors">
    Ensure your database is running and credentials are correct. Check `application.properties` in each microservice:
    
    ```properties
    spring.datasource.url=jdbc:postgresql://localhost:5432/db_pacientes
    spring.datasource.username=your_username
    spring.datasource.password=your_password
    ```
  </Accordion>
  
  <Accordion title="Frontend Proxy Errors">
    The frontend uses Vite proxy to communicate with microservices. Ensure all backend services are running before starting the frontend. Check `nova-frontend/vite.config.ts:30` for proxy configuration.
  </Accordion>
  
  <Accordion title="Feign Client Errors">
    If you see Feign client connection errors, verify that all dependent microservices are running. For example, `msvc-cita` requires `msvc-paciente`, `msvc-medico`, and `msvc-diagnostico` to be running.
  </Accordion>
</AccordionGroup>
