## Introduction

The Doctor Service (msvc-medico) is a microservice that manages all doctor-related operations in the NOVA.ing Atención Médica platform. It provides endpoints for doctor registration, profile management, appointment scheduling, and diagnostic record creation.

## Base URL

```
http://localhost:8080/medicos
```

## Key Features

- **Doctor Management**: Create, read, update, and delete doctor profiles
- **Appointment Access**: Retrieve all appointments for a specific doctor
- **Appointment Scheduling**: Schedule new appointments via doctor service
- **Diagnostic Recording**: Register diagnostic records for patient consultations
- **Soft Delete Support**: Logical deletion with permanent delete option
- **Schedule Management**: Manage doctor availability schedules by day of week

## Data Model

### Doctor Entity

The core doctor entity contains:

<ResponseField name="id" type="Long">
  Unique identifier (auto-generated)
</ResponseField>

<ResponseField name="nombres" type="string" required>
  Doctor's first name(s)
</ResponseField>

<ResponseField name="apellidos" type="string" required>
  Doctor's last name(s)
</ResponseField>

<ResponseField name="especialidad" type="enum" required>
  Medical specialty (see Doctor Specialties above)
</ResponseField>

<ResponseField name="telefono" type="string" required>
  Phone number (format: 9XXXXXXXX - must start with 9 and have 9 digits)
</ResponseField>

<ResponseField name="email" type="string" required>
  Email address (unique)
</ResponseField>

<ResponseField name="dni" type="string" required>
  National ID number (format: 8 digits, cannot start with 0)
</ResponseField>

<ResponseField name="estado" type="enum" required>
  Current status (ACTIVO or INACTIVO)
</ResponseField>

<ResponseField name="horarios" type="array">
  List of doctor's working schedules
  <Expandable title="HorarioMedico properties">
    <ResponseField name="id" type="Long">
      Schedule entry ID
    </ResponseField>
    <ResponseField name="diaSemana" type="enum">
      Day of week (MONDAY, TUESDAY, WEDNESDAY, THURSDAY, FRIDAY, SATURDAY, SUNDAY)
    </ResponseField>
    <ResponseField name="horaInicio" type="LocalTime">
      Start time (format: HH:mm:ss)
    </ResponseField>
    <ResponseField name="horaFin" type="LocalTime">
      End time (format: HH:mm:ss)
    </ResponseField>
  </Expandable>
</ResponseField>

<ResponseField name="usuarioId" type="Long">
  Associated user account ID (for authentication)
</ResponseField>

## Validation Rules

### Phone Number
- Must start with 9
- Must be exactly 9 digits
- Pattern: `^9[0-9]{8}$`

### DNI (National ID)
- Must be exactly 8 digits
- Cannot start with 0
- Pattern: `^[1-9][0-9]{7}$`

### Required Fields
- All fields marked as required must not be blank
- Unique constraints apply to: telefono, email, dni
