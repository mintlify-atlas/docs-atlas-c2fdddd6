## Overview

The NOVA.ing frontend is a modern React application built with Vite, TypeScript, and Tailwind CSS. It provides the user interface for managing patients, doctors, appointments, and diagnoses through a responsive web application.

## Technology Stack

- **React 18.3** - UI framework
- **TypeScript 5.8** - Type-safe JavaScript
- **Vite 6.3** - Fast build tool and dev server
- **Tailwind CSS 3.4** - Utility-first CSS framework
- **React Router 7.3** - Client-side routing
- **Axios 1.13** - HTTP client for API calls
- **Zustand 5.0** - State management
- **React Hook Form 7.71** - Form handling

## Prerequisites

Before setting up the frontend, ensure you have:

- **Node.js 18+** installed
- **npm** or **yarn** package manager
- All backend microservices running (ports 8080-8084)
- Apache Jena Fuseki running (port 3030) for semantic features

### Verify Node.js Installation

```bash
node -v  # Should show v18 or higher
npm -v   # Should show npm version
```

## Frontend Architecture

The frontend communicates with backend microservices through Vite's proxy configuration:

| API Route | Backend Service | Port |
|-----------|----------------|------|
| `/api/pacientes` | msvc-paciente | 8083 |
| `/api/medicos` | msvc-medico | 8080 |
| `/api/citas` | msvc-cita | 8081 |
| `/api/diagnosticos` | msvc-diagnostico | 8082 |
| `/api/v1/semantic` | msvc-web-semantica | 8084 |

## Build for Production

### Step 1: Type Check

Verify TypeScript types before building:

```bash
npm run check
```

This runs `tsc -b --noEmit` to check for type errors without generating output.

### Step 2: Build Production Bundle

Create an optimized production build:

```bash
npm run build
```

This command:
1. Runs TypeScript compiler (`tsc -b`)
2. Builds the application with Vite
3. Generates optimized output in `dist/` directory

<Note>
If you encounter build errors, try building first before running the dev server:
```bash
npm run build && npm run dev
```
</Note>

### Step 3: Preview Production Build

Test the production build locally:

```bash
npm run preview
```

This serves the `dist/` directory on `http://localhost:4173`

## Configuration

### Vite Configuration

The frontend uses Vite for development and building. Configuration is in `vite.config.ts`:

```typescript vite.config.ts
import { defineConfig } from 'vite'
import react from '@vitejs/plugin-react'
import tsconfigPaths from "vite-tsconfig-paths";

export default defineConfig({
  build: {
    sourcemap: 'hidden',
  },
  plugins: [
    react({
      babel: {
        plugins: [
          'react-dev-locator',
        ],
      },
    }),
    tsconfigPaths()
  ],
  server: {
    proxy: {
      '/api/pacientes': {
        target: 'http://localhost:8083',
        changeOrigin: true,
        rewrite: (path) => path.replace(/^\/api\/pacientes/, '/pacientes'),
      },
      '/api/medicos': {
        target: 'http://localhost:8080',
        changeOrigin: true,
        rewrite: (path) => path.replace(/^\/api\/medicos/, '/medicos'),
      },
      '/api/citas': {
        target: 'http://localhost:8081',
        changeOrigin: true,
        rewrite: (path) => path.replace(/^\/api\/citas/, '/citas'),
      },
      '/api/diagnosticos': {
        target: 'http://localhost:8082',
        changeOrigin: true,
        rewrite: (path) => path.replace(/^\/api\/diagnosticos/, '/diagnosticos'),
      },
      '/api/v1/semantic': {
        target: 'http://localhost:8084',
        changeOrigin: true,
      },
    },
  },
})
```

### Proxy Configuration Explained

The Vite proxy rewrites API requests during development:

1. Frontend makes request to: `http://localhost:5173/api/pacientes/list`
2. Vite proxy forwards to: `http://localhost:8083/pacientes/list`
3. Response returns to frontend

This avoids CORS issues during development.

### Environment Variables

Create environment-specific configuration files:

<CodeGroup>
```env .env.development
# Development environment
VITE_API_BASE_URL=http://localhost
VITE_BACKEND_PORT_MEDICO=8080
VITE_BACKEND_PORT_CITA=8081
VITE_BACKEND_PORT_DIAGNOSTICO=8082
VITE_BACKEND_PORT_PACIENTE=8083
VITE_BACKEND_PORT_SEMANTIC=8084
```

```env .env.production
# Production environment
VITE_API_BASE_URL=https://api.nova.ing
VITE_BACKEND_PORT_MEDICO=443
VITE_BACKEND_PORT_CITA=443
VITE_BACKEND_PORT_DIAGNOSTICO=443
VITE_BACKEND_PORT_PACIENTE=443
VITE_BACKEND_PORT_SEMANTIC=443
```
</CodeGroup>

Access environment variables in code:

```typescript
const apiUrl = import.meta.env.VITE_API_BASE_URL;
const medicoPort = import.meta.env.VITE_BACKEND_PORT_MEDICO;
```

<Warning>
All Vite environment variables must be prefixed with `VITE_` to be exposed to the client code.
</Warning>

## Production Deployment

### Option 1: Static Hosting (Nginx)

<Steps>
  <Step title="Build the Application">
    ```bash
    npm run build
    ```
    Output is generated in `dist/` directory
  </Step>

  <Step title="Configure Nginx">
    Create nginx configuration:
    ```nginx /etc/nginx/sites-available/nova-frontend
    server {
        listen 80;
        server_name nova.ing www.nova.ing;
        root /var/www/nova-frontend/dist;
        index index.html;

        # Handle client-side routing
        location / {
            try_files $uri $uri/ /index.html;
        }

        # Proxy API requests to backend
        location /api/pacientes/ {
            proxy_pass http://localhost:8083/pacientes/;
            proxy_http_version 1.1;
            proxy_set_header Upgrade $http_upgrade;
            proxy_set_header Connection 'upgrade';
            proxy_set_header Host $host;
            proxy_cache_bypass $http_upgrade;
        }

        location /api/medicos/ {
            proxy_pass http://localhost:8080/medicos/;
            proxy_http_version 1.1;
            proxy_set_header Upgrade $http_upgrade;
            proxy_set_header Connection 'upgrade';
            proxy_set_header Host $host;
            proxy_cache_bypass $http_upgrade;
        }

        location /api/citas/ {
            proxy_pass http://localhost:8081/citas/;
            proxy_http_version 1.1;
            proxy_set_header Upgrade $http_upgrade;
            proxy_set_header Connection 'upgrade';
            proxy_set_header Host $host;
            proxy_cache_bypass $http_upgrade;
        }

        location /api/diagnosticos/ {
            proxy_pass http://localhost:8082/diagnosticos/;
            proxy_http_version 1.1;
            proxy_set_header Upgrade $http_upgrade;
            proxy_set_header Connection 'upgrade';
            proxy_set_header Host $host;
            proxy_cache_bypass $http_upgrade;
        }

        location /api/v1/semantic/ {
            proxy_pass http://localhost:8084/api/v1/semantic/;
            proxy_http_version 1.1;
            proxy_set_header Upgrade $http_upgrade;
            proxy_set_header Connection 'upgrade';
            proxy_set_header Host $host;
            proxy_cache_bypass $http_upgrade;
        }

        # Cache static assets
        location ~* \.(js|css|png|jpg|jpeg|gif|ico|svg|woff|woff2|ttf|eot)$ {
            expires 1y;
            add_header Cache-Control "public, immutable";
        }
    }
    ```
  </Step>

  <Step title="Deploy Files">
    ```bash
    # Copy dist files to web server
    sudo mkdir -p /var/www/nova-frontend
    sudo cp -r dist/* /var/www/nova-frontend/
    sudo chown -R www-data:www-data /var/www/nova-frontend
    ```
  </Step>

  <Step title="Enable Site and Restart Nginx">
    ```bash
    sudo ln -s /etc/nginx/sites-available/nova-frontend /etc/nginx/sites-enabled/
    sudo nginx -t  # Test configuration
    sudo systemctl restart nginx
    ```
  </Step>
</Steps>

### Option 2: Docker Container

Create a Dockerfile for containerized deployment:

```dockerfile Dockerfile
# Build stage
FROM node:18-alpine AS build
WORKDIR /app

# Copy package files
COPY package*.json ./
RUN npm ci

# Copy source files and build
COPY . .
RUN npm run build

# Production stage
FROM nginx:alpine
COPY --from=build /app/dist /usr/share/nginx/html

# Copy nginx configuration
COPY nginx.conf /etc/nginx/conf.d/default.conf

EXPOSE 80
CMD ["nginx", "-g", "daemon off;"]
```

Build and run the Docker container:

```bash
# Build image
docker build -t nova-frontend .

# Run container
docker run -d -p 80:80 --name nova-frontend nova-frontend
```

## Troubleshooting

### Port 5173 Already in Use

**Problem**: `Error: Port 5173 is already in use`

**Solution**: Specify a different port:
```bash
npm run dev -- --port 5174
```

### Backend Connection Refused

**Problem**: Frontend cannot connect to backend services

**Solution**:
1. Verify all microservices are running:
   ```bash
   curl http://localhost:8080/actuator/health
   curl http://localhost:8081/actuator/health
   curl http://localhost:8082/actuator/health
   curl http://localhost:8083/actuator/health
   curl http://localhost:8084/actuator/health
   ```
2. Check Vite proxy configuration in `vite.config.ts`
3. Ensure no firewall blocking local ports

### Build Fails with Type Errors

**Problem**: TypeScript compilation errors during build

**Solution**:
1. Run type check to see all errors:
   ```bash
   npm run check
   ```
2. Fix type errors in the reported files
3. Alternatively, build without type checking (not recommended):
   ```bash
   vite build
   ```

### Module Not Found Errors

**Problem**: `Error: Cannot find module 'axios'` or similar

**Solution**:
1. Delete `node_modules` and reinstall:
   ```bash
   rm -rf node_modules package-lock.json
   npm install
   ```
2. Clear npm cache if issues persist:
   ```bash
   npm cache clean --force
   npm install
   ```

### Blank Page After Deployment

**Problem**: Production build shows blank page

**Solution**:
1. Check browser console for errors (F12)
2. Verify `index.html` is served for all routes
3. Check nginx/server configuration for client-side routing:
   ```nginx
   location / {
       try_files $uri $uri/ /index.html;
   }
   ```
4. Verify base path in `vite.config.ts` if deploying to subdirectory:
   ```typescript
   export default defineConfig({
     base: '/nova/',  // If deploying to example.com/nova/
   })
   ```

### API Requests Fail in Production

**Problem**: API calls return 404 or CORS errors in production

**Solution**:
1. Update API base URLs in production build
2. Configure CORS on backend services:
   ```java
   @CrossOrigin(origins = "https://nova.ing")
   ```
3. Ensure nginx proxy configuration matches API routes
4. Check browser network tab (F12) for actual request URLs

## Performance Optimization

### Code Splitting

Vite automatically splits code by route. Optimize further with dynamic imports:

```typescript
// Lazy load components
const PatientList = lazy(() => import('./components/PatientList'));

// Use with Suspense
<Suspense fallback={<Loading />}>
  <PatientList />
</Suspense>
```

### Build Optimization

Configure build options in `vite.config.ts`:

```typescript
export default defineConfig({
  build: {
    sourcemap: false,  // Disable source maps for production
    minify: 'terser',  // Use terser for better minification
    terserOptions: {
      compress: {
        drop_console: true,  // Remove console.log
      },
    },
    rollupOptions: {
      output: {
        manualChunks: {
          vendor: ['react', 'react-dom', 'react-router-dom'],
          forms: ['react-hook-form'],
          state: ['zustand'],
        },
      },
    },
  },
})
```

### Asset Optimization

1. **Compress images** before adding to project
2. **Use WebP format** for better compression
3. **Lazy load images** below the fold
4. **Enable gzip/brotli** compression on server
